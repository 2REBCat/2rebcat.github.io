정적 사이트는 단순해 보여도 배포 전에 꼭 확인해야 하는 항목이 있습니다.

## 내가 먼저 보는 항목

1. 첫 화면이 의도한 페이지인지
2. 메뉴 링크가 실제 구조와 맞는지
3. 이미지나 아이콘 파일이 중복 없이 정리되어 있는지
4. 소개와 기록의 역할이 섞이지 않는지

## 왜 필요한가

작은 누락이 바로 사용자 경험 저하로 이어지기 때문입니다. 특히 GitHub Pages처럼 정적 배포가 중심인 구조에서는 파일 경로 관리가 매우 중요합니다.
